Lists log service indexes associated with a log service.

Scopes

You will need authorization for the https://www.googleapis.com/auth/cloud-platform scope to make a valid call.

If unset, the scope for this method defaults to https://www.googleapis.com/auth/cloud-platform. You can set the scope for this method like this: logging1-beta3 --scope <scope> projects log-services-indexes-list ...

Required Scalar Arguments

  • <projects-id> (string)
    • Part of serviceName. A log service resource of the form /projects/*/logServices/*. The service indexes of the log service are returned. Example: &#34;/projects/myProj/logServices/appengine.googleapis.com&#34;.
  • <log-services-id> (string)
    • Part of serviceName. See documentation of projectsId.

Optional Output Flags

The method's return value a JSON encoded structure, which will be written to standard output by default.

  • -o out
    • out specifies the destination to which to write the server's result to. It will be a JSON-encoded structure. The destination may be - to indicate standard output, or a filepath that is to contain the received bytes. If unset, it defaults to standard output.

Optional Method Properties

You may set the following properties to further configure the call. Please note that -p is followed by one or more key-value-pairs, and is called like this -p k1=v1 k2=v2 even though the listing below repeats the -p for completeness.

  • -p page-token=string

    • An opaque token, returned as nextPageToken by a prior ListLogServiceIndexes operation. If pageToken is supplied, then the other fields of this request are ignored, and instead the previous ListLogServiceIndexes operation is continued.

  • -p log=string

    • A log resource like /projects/project_id/logs/log_name, identifying the log for which to list service indexes.

  • -p index-prefix=string

    • Restricts the indexes returned to be those with a specified prefix. The prefix has the form &#34;/label_value/label_value/...&#34;, in order corresponding to the [LogService indexKeys][google.logging.v1.LogService.index_keys]. Non-empty prefixes must begin with / . Example prefixes: + &#34;/myModule/&#34; retrieves App Engine versions associated with myModule. The trailing slash terminates the value. + &#34;/myModule&#34; retrieves App Engine modules with names beginning with myModule. + &#34;&#34; retrieves all indexes.

  • -p depth=integer

    • A limit to the number of levels of the index hierarchy that are expanded. If depth is 0, it defaults to the level specified by the prefix field (the number of slash separators). The default empty prefix implies a depth of 1. It is an error for depth to be any non-zero value less than the number of components in indexPrefix.

  • -p page-size=integer

    • The maximum number of log service index resources to return in one operation.
